# Editing BlastoDB directly on GitHub

You can edit BlastoDB directly on GitHub, without needing to download the repository locally. Here you find a short overview on what you need to change the website, and how to make these small changes.

However, this has the disadvantage that there is no preview of your changes before building and deploying the website. For this reason, I recommend you only use this for smaller changes, like adding, removing or editing:
- a line of text in an existing file
- publidations
- news
- lab protocols

If you wand to do larger changes, I recommend to [edit BlastoDB locally], so that you can see the changes. An easy rule-of-thumb here is:
- Do you only write or change text, essentially [normal markdown](todo: link)* -> Edit directly on GitHub
- Do you wand to create buttons, visual elements or interactions with [custom elements](todo: link) -> Edit locall, use [this tutorial](todo: link).

*An exception to this would be the creation of punlications, news or lab protocols, as they can easily be created and are automatically loaded onto the website.

## What you Need to Change the Website

Before you can make changes, you need:
1. A *GitHub account*. Create one here [https://github.com/](https://github.com/).
2. You need to be a *contributor* in the repository of BlastoDB. Ask somone with access to add you to the repository.

BlastoDB is written in Markdown (TODO: link). Markdown is a simple markup language, that has a few simple commands to style text. If you are unfamiliar with it, I recommend you to check out [this overview](TODO: link).
[Here](https://www.markdownguide.org/cheat-sheet/) you also find a small cheat-sheet with all commands.
